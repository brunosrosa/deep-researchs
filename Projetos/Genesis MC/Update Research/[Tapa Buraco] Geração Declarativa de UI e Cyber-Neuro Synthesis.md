🔍 PROMPT DE DEEP RESEARCH 2: Geração Declarativa de UI e Cyber-Neuro Synthesis

**[COPIE A PARTIR DAQUI]**

ATUE COMO UM ARQUITETO FRONTEND MASTER, ESPECIALISTA EM INTERAÇÃO HUMANO-MÁQUINA (IHC) E PESQUISADOR DE NEURODESIGN (FOCO NO ESTADO DA ARTE DE 2025/2026).

**CONTEXTO MESTRE (PROJETO GENESIS MC - SODA):** Estamos desenvolvendo o SODA (Sovereign Operating Data Architecture), um Sistema Operacional Agêntico local "bare-metal" (núcleo Rust, interface Tauri v2). O frontend em React (utilizando Tldraw/Xyflow e Shadcn UI) é ESTRITAMENTE PASSIVO. Ele não possui regras de negócios, apenas escuta eventos binários (IPC Zero-Copy) do Rust e desenha a tela. O usuário alvo possui perfil 2e/TDAH. A estética exigida é a "Cyber-Neuro Synthesis": alta densidade de informação, design utilitário monocromático ("Nothing Design"), feedback mecânico quase instantâneo (<50ms) e Zero Layout Shifts.

**O PROBLEMA A SER RESOLVIDO (ALUCINAÇÃO VISUAL E FLOW-DEBT):** Quando pedimos para agentes de IA (via "Vibe Design" ou "Vibe Coding") criarem interfaces para novas funcionalidades, eles tendem a homogeneizar o design: geram UIs lentas, componentes genéricos com sombras pesadas (estilo SaaS comercial), e tentam injetar lógicas pesadas de JavaScript/DOM no frontend, quebrando nossa arquitetura passiva e gerando "Flow-Debt" (Dívida de Fluxo).

**A MISSÃO DE PESQUISA:** Realize uma pesquisa aprofundada na literatura científica e em soluções pragmáticas _open-source_ de 2025/2026 sobre **Generative UI (GenUI)**, **Protocolo A2UI (Agent-to-UI)** e renderização declarativa em ecossistemas multiagentes. Descubra como transmutar ideias abstratas saídas de um Brainstorm (Fase 0) para um documento `DESIGN.md` estruturado que o frontend passivo possa renderizar com segurança.

Entregue um relatório técnico focado nos seguintes pontos:

- **O Protocolo A2UI / UI Declarativa:** Como a vanguarda estrutura a comunicação onde o Agente LLM não escreve código React livremente, mas sim emite "Árvores de Intenção" (schemas JSON ou definições paramétricas) que um frontend pré-compilado lê e converte em componentes seguros na tela? (Investigue padrões como o A2UI do Google, renderizadores JSON-para-React e arquiteturas Server-Driven UI adaptadas para agentes locais).- **Engenharia de Prompt para Neurodesign (Cyber-Neuro Synthesis):** Quais são as restrições arquiteturais matemáticas (Constraint-Based Design) que devemos impor aos agentes geradores para proibir alucinações estéticas e forçar o uso de "Vidro Tátil" (Glassmorphism funcional), "Instância Mecânica" e "Previsibilidade Espacial"?- **Resolução de Conflitos e "First Draft Protocol":** Como frameworks modernos resolvem o embate entre um "Agente de Design" (que quer uma UI complexa) e o "Agente de Código" (que exige eficiência no backend Tauri/Rust) antes da aprovação final?

**FORMATO DE SAÍDA EXIGIDO:** Gere um documento analítico arquitetural. Sem código JavaScript verboso ou configurações de Node.js. Quero o diagrama lógico da ponte A2UI, as regras de _schema_ recomendadas, e as diretrizes heurísticas exatas que formarão a nossa "Super Skill" de Design (`@vibe-designer`). O objetivo é garantir que o agente desenhe UIs utilitárias de vanguarda que aliviem a carga cognitiva do TDAH, em vez de sobrecarregá-la.
